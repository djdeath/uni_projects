Rapport de projet de Système






Dans ce projet, il nous tait demandé d'écrire un petit jeu dans lequel le but est de taper le plus vite possible les mots qui apparaissent l'écran.

Ce projet de compose de deux programmes un client et un serveur. Le rôle du client est d'afficher les mots envoyés par le serveur et d'envoyer au serveur les mots que le joueur tape. Le serveur reçoit les mots taper par le client et en envoie d'autres en fonction de certains paramètres.




  1. Le serveur :




Le programme serveur dispose des fonctionnalites suivantes :

  • Gestion des sockets

  • Gestion des équipes

  • Gestion des messages

  • Envois de séquences d'échappement ANSI (coloration des mots)

  • Un fichier de configuration contenant les différentes options du jeu

  • Une console permettant de lancer plusieurs serveurs depuis le même programme, ainsi que changer certaines options la volée



    1. Fichier de configuration :



Le programme serveur a besoin pour fonctionner d'un fichier de configuration contenant toutes les valeurs d'initialisation d'une partie. Ce fichier doit s'appeler sv.cfg et doit se trouver dans le répertoire depuis lequel on lance le programme.

Voici un exemple de fichier de configuration :


# commentaire


sv_max_clients = 10

sv_entry_msg = Hello ;)


game_width = 100

game_height = 15



game_limit = 100


game_type = 0


game_speed = 0.5

game_accel = 0.02


game_accel_timeout = 7.0


dict_file = dict.txt

dict_num = 10000



Détail des options :


sv_max_clients : nombre maximum de clients

sv_entry_msg : Message d'entre sur le serveur (envoyé aux clients lors de la connexion)

game_width : largeur de la table de jeu

game_height : hauteur de la table de jeu

game_limit : nombre de mots avant la fin du jeu

game_type : type de jeu (0 pour statique, 1 pour dynamique)

game_speed : vitesse du jeu (en mots par secondes)

game_accel : acclration du jeu

game_accel_timeout : intervalle de temps au bout du quel on recalcule la vitesse du jeu

dict_file : nom du fichier du dictionnaire (position relative au répertoire courant)

dict_num : nombre de mots charger depuis le fichier dictionnaire.


Les options sv_max_clients, sv_entry_msg et game_limit peuvent être changées à la volée pendant qu'un serveur fonctionne. Les options game_speed, game_accel, game_accel_timeout, et game_type nécessite de stopper le serveur et de le redémarrer (sv_stop sv_start) pour prendre effet. Ce qui n'implique pas de déconnexion des clients. Enfin les options dict_file et dict_num nécessite de détruire le serveur et d'en recréer un autre (sv_destroy sv_create).



    1. Utilisation :



Lorsque l'on exécute le programme serveur, aucun serveur n'est pas lancé par défaut. On se trouve sur une console permettant d'exécuter des commandes. Voici la liste des commandes :


quit : quitter le programme

help / list : afficher la liste des commandes

sv_prev / sv_next : passer au serveur précédent / suivant

sv_create : lancer un nouveau serveur (le port doit être précisé)

sv_destroy : détruire le serveur courant

sv_start : lancer la partie sur le serveur courant

sv_stop : stopper le serveur sur le serveur courant

sv_pause : mettre le serveur en pause

sv_unpause : enlever le serveur du mode pause

sv_list : liste des clients sur le serveur

sv_kick : déconnecter un client (il faut préciser son nom)

sv_get : afficher la valeur d'une option du serveur (il faut préciser l'option)

sv_get : positionner la valeur d'une option (il faut préciser l'option et la valeur)


Lorsqu'un serveur est mis en pause, aucun des clients ne pourra taper de mot. La destruction d'un serveur par la commande sv_destroy ainsi que la commande quit ferme correctement tous les descripteurs ouverts.

Voici une séquence de commandes permettant de lancer deux serveurs, l'un sur le port 2044 et l'autre sur le port 1983, puis de mettre en pause le serveur sur le port 2044 :


sv_create 2044

sv_start

sv_create 1983

sv_start

sv_prev

sv_pause



    1. Appels systèmes et fonctionnement du programme :



Notre programme serveur est basé sur un système d'évènement (event.c/h). Il agit en fonction des évènements qui se produisent sur un ensemble de descripteurs mais aussi en fonction du temps qui s'écoule. Par exemple, pour être prévenues des éventuels changement d'état sur un descripteur, les autres parties du programme doivent s'enregistrer au près de la boucle d'évènement pour en être prévenue par un mécanisme de fonction de rappel.


Voici la liste des appels systèmes utilisés par notre programme serveur :

  • open / read / write / close

  • poll (ou select selon la disponibilité du système)

  • gettimeofday

  • socket / bind / listen / accept / shutdown



    1. Limitations :



Notre programme utilise un algorithme de placement de mots naïf. On tire d'abord un mot au hasard dans le dictionnaire. Ce mot ne doit pas être présent sur la grille de jeu. On prend ensuite une ligne au hasard, et on essaye de trouver un emplacement pour positionner ce mot sur cette ligne en partant de la gauche. Chacune de ses opération sera répétée trois fois. Si au bout de 3 essais on ne parvient pas à trouver de position satisfaisante le programme réessayera alors plus tard. Cet algorithme a deux inconvénients majeurs ; il n'assure pas qu'à chaque passage un mot sera placé sur la grille, il a tendance à accumuler les mots sur le côté gauche de la grille.




  1. Le client :




Le client que nous avons écrit utilise la librairie graphique GTK+ 2. Les fonctionnalités qu'il implémente sont les suivantes :

  • Fichier de configuration (permet la sauvegarde des options)

  • Affichage des mots en couleur

  • Affichage du score des équipes et des joueurs

  • Fenêtre de chat (permettant de communiquer avec les autres joueurs)



    1. Fichier de configuration :



Le fichier de configuration doit se trouver dans le répertoire courant et son nom doit être cl.cfg. Il n'est pas nécessaire au chargement du programme. Il sera recréé à la sortie du programme avec les valeurs rentrées par l'utilisateur. Voici un exemple de fichier de configuration :


# COMMENT

sv_port = 2044

name = Dj-Death

sv_host = localhost

team = ATR



Détail des options :


sv_host : adresse du serveur auquel l'on doit se connecter

sv_port : port du serveur

name : nom du joueur

team : nom de l'équipe du joueur



    1. Utilisation :


Le client se compose de trois fenêtres, une pour jouer (taper les mots), une pour afficher des scores des équipes et des joueurs, et enfin une dernière pour afficher les messages. Lorsque l'utilisateur souhaite se connecter à un serveur, il doit aller dans le menu Fichier puis Se Connecter et entrer l'adresse du serveur auquel il souhaite se connecter ainsi que le port. Si il n'a pas précisé son nom, une fenêtre s'ouvrira pour qu'il tape son nom ainsi que son équipe. L'utilisateur pourra ensuite taper les mots qui apparaissent à l'écran.




  1. Compilation :




Notre projet se compile de la manière suivante ; il faut tout d'abord se placer dans le répertoire des sources puis faire :


$ ./configure

$ make
